RAG 学习笔记 · 1.2 参数记忆 vs 非参数记忆——RAG 的核心解法

章节定位: 1.1 我们聊了 LLM 三大缺陷(幻觉 / 知识截止 / 上下文窗口),本节看 RAG 怎么从原理上解决这些缺陷——两种”记忆”的区别。

一、两种”记忆”方式

想象一下你学习一门新知识,有两种方式:

  • 死记硬背:把整本教材背下来,考试时从脑子里”回忆”
  • 开卷考试:考试时翻书、查资料,只需要会”用”资料

LLM 也有这两种方式,只是换了个名字:

维度 参数记忆(Parametric Memory) 非参数记忆(Non-Parametric Memory)
存储在哪 模型参数里(几百亿到几万亿个数字) 外部数据库、向量库、文件系统
怎么获取 前向推理时激活相关参数 检索时查询、读取、拼到 prompt 里
更新成本 重训或微调(几十万到几亿美元) 增删文档或重建索引(几分钱)
更新速度 月级到年级 实时(秒级)
容量上限 受限于模型大小 几乎无限(取决于磁盘)
可解释性 差(黑盒) 强(可追溯到具体文档)
知识粒度 模糊(分布式的) 精确(可逐条引用)

🎯 一句话总结:RAG 的本质是给 LLM 装一个**”外挂大脑”**,让 LLM 在自己”脑子”之外,还能”翻资料”。

二、为什么 LLM 只有”参数记忆”会出问题?

1. 存储密度低

GPT-4 估计有 1.8 万亿参数,GLM-5 是 744B(激活 40B)。听起来很大,但实际能”记住”的信息量远小于训练数据量。

训练数据是几十 TB 的清洗后文本,压缩到几百 GB 的参数里,压缩比超过 50:1——大量细节被丢掉了:

  • 罕见的人名、地名、术语
  • 长尾事实(冷门的历史事件、小众的学术引用)
  • 精确的数字、日期、版本号
  • 私有知识(企业内部文档、个人笔记)

2. 写入代价极高

想让模型”记住”一个新事实,比如”GLM-5.2 在 2026 年 6 月 17 日发布”:

  • 准备相关训练数据
  • 走一遍完整的 fine-tuning 或 continued pretraining
  • 几万 GPU 小时
  • 几万到几百万美元

而同样的信息,用 RAG 只需要往向量库里加一条文档,几秒钟搞定。

3. 检索不可控

  • 模型”记住”了什么?——只有 LLM 自己的 attention 模式知道,人类无从知晓
  • 模型会”忘记”或”混淆”相似的知识(比如把 GLM-5 和 GLM-5.2 搞混)
  • 难以做精确的引用和溯源——幻觉就是这么来的

三、非参数记忆怎么”补”这些短板?

RAG 的核心机制:把”知识”和”推理”解耦。

一个直觉化的比喻

原来:让一个人背下整本百科全书(参数化)

前者耗时长、记得不牢、还容易记错;后者省力、随时查新东西、还能精确引用。

四、实际工作流的对比

场景:问 “GLM-5.2 是什么时候发布的?”

纯参数记忆(传统 LLM):

→ 容易说错,因为是从模糊的”参数分布”中猜的。

纯非参数记忆(纯 RAG,无 LLM 推理):

→ 准确但机械,无法做总结、对比、推理。

RAG(参数记忆 + 非参数记忆协同):

→ 既准确,又能综合、推理、对比、总结。

这就是 RAG 的魔力——**不是替代 LLM 的能力,而是给 LLM 一个可靠的”参考资料库”**。

五、RAG 原始论文的核心论点(2020)

2020 年 Patrick Lewis 等人提出 RAG 时,论文的核心论点就一句话:

“RAG combines a pre-trained retriever (non-parametric memory) with a pre-trained seq2seq generator (parametric memory).”

也就是显式地把两种记忆融合:

  • Retriever(DPR):基于密集向量检索,负责非参数记忆
  • Generator(BART):基于参数化的 seq2seq,负责生成

后续的所有 Advanced RAG、Agentic RAG、GraphRAG,本质上都在优化这个双系统架构的某个环节。

六、2026 年视角:两种记忆的边界正在模糊

1. 参数侧:模型越来越大,记忆能力越来越强

  • GPT-4:1.8T 参数
  • GLM-5:744B(激活 40B),训练数据 28.5T tokens
  • 下一代模型传闻 10T+

更大的参数能记住更多事实,**但仍然解决不了”私有数据 100% 缺失”**——企业内文档、个人笔记、行业资料,模型从训练阶段就完全没见过。

2. 非参数侧:检索能力越来越强

时期 主要技术 特点
早期(~2018) BM25 关键词匹配 精确但无法语义理解
2020-2023 向量检索(DPR、Contriever) 语义匹配,但有 lost in the middle
2023-2025 Hybrid Search(BM25 + 向量)+ Reranker 召回+精排两阶段
2025-2026 Agentic RAG、GraphRAG、ColBERT 检索成为行动循环的一部分

3. 融合侧:长期记忆系统(Memory-Augmented AI)

2026 年最热的方向是**”让 AI 拥有持续记忆”**:

  • AI 不再每次重新检索,而是形成用户画像、记录历史决策、持续更新知识状态
  • 代表工作:MemVerse(多模态记忆)、ACE(Agentic Context Engineering)
  • 这让 RAG 从”外部知识补丁”变成”AI 认知结构的一部分”

七、本节核心要点

  1. 两种记忆的本质区别:参数记忆内化在模型里,非参数记忆存储在外部;前者快但难更新,后者慢但灵活。
  2. RAG 不是替代 LLM,而是给 LLM 装”外挂大脑”:LLM 不用”学”所有知识,只需要”会读 + 会用”资料。
  3. 2026 年的双记忆协同:参数侧持续变大(更强的内化知识),非参数侧持续变强(GraphRAG、Agentic),融合侧出现长期记忆系统。
  4. RAG 解决的是结构性问题:企业私有数据 100% 缺失、知识截止的物理限制、长尾事实的存储密度问题——这些不是更大模型能解决的,必须靠非参数记忆补。
